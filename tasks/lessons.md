# Lessons

## 2026-04-26

- For small MVP projects, keep the discovery interview tightly focused on the core feature set instead of asking enterprise-scale product questions.
- When the user asks for interview first and spec approval before implementation, stop at documentation and do not begin coding early.
- Security-sensitive scope items such as internal booking access should be clarified explicitly instead of inferred.
- For booking systems, concurrency control, timezone storage, delete semantics, and retry behavior should be specified in the spec before implementation starts.
